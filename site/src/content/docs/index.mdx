## Two Modernization Paths

<div class="section-lede">
  Every workload needs a deliberate path. Some workloads should move quickly and
  become better operated on Azure. Others deserve deeper transformation because
  the business needs agility, scale, analytics, or AI readiness.
</div>

<div class="path-grid">
  <article class="path-card path-card--stabilize">
    <div class="eyebrow">Path 1</div>
    <h3>Stabilize</h3>
    <div class="path-card-text">
      Move priority workloads to Azure, reduce datacenter dependency, right-size
      infrastructure, and prepare operational data for governed analytics.
    </div>
    <a href="/dc2fabric/horizons/h1-lift-shift/">See the stabilize path</a>
  </article>
  <article class="path-card path-card--transform">
    <div class="eyebrow">Path 2</div>
    <h3>Transform</h3>
    <div class="path-card-text">

      Fabric integration, governance, and AI-ready foundations.
    </div>
    <a href="/dc2fabric/horizons/h2-modernize/">See the transform path</a>
  </article>
</div>

## Fabric as the Destination

<section
  class="fabric-panel"
  aria-label="Fabric as the governed analytics destination"
>
  <div>
    <div class="eyebrow">Governed analytics</div>
    <h3>Connect migration outcomes to Fabric value</h3>
    <div class="fabric-copy">
      SQL Managed Instance and Azure SQL Database can mirror supported
      operational data into Microsoft Fabric, where it can become governed data
      products for Power BI, machine learning, real-time intelligence, and AI
      scenarios.
    </div>
  </div>
  <ul>
    <li>
      Validate source support, permissions, networking, and capacity before
      committing timelines.
    </li>
    <li>
      Recreate Fabric security, ownership, labels, endorsement, lineage, and
      access reviews explicitly.
    </li>
    <li>
      Treat Fabric as the analytics operating model, not just a replication
      target.
    </li>
  </ul>
</section>
